In this section, we will access the **Amazon CloudWatch Dashboard** to monitor the status and performance of AWS resources deployed in the system.

The CloudWatch Dashboard provides a centralized interface for viewing **Metrics**, **Logs**, and **Alarms** collected from AWS services. It enables administrators to monitor system health and quickly identify potential issues.

---

## Step 1. Open the Amazon CloudWatch Service

Sign in to the **AWS Management Console**.

Perform the following steps:

1. Search for **Amazon CloudWatch**.
2. Select **CloudWatch**.
3. Open the **Dashboard** page.

---

## Step 2. Access the Dashboard

After opening the Dashboard, you can view the following information:

- Available CloudWatch Dashboards.
- Monitoring charts and graphs.
- Collected system metrics.
- The status of existing CloudWatch Alarms.

The dashboard provides a centralized view for monitoring the overall health of the AWS environment.

## Step 3. Verify the Dashboard

After successfully opening the dashboard:

- Verify that the dashboard is displayed correctly.
- Review the available metrics.
- Check the status of existing alarms (if any).
- Familiarize yourself with the CloudWatch management interface.

The dashboard will be used throughout the following sections to monitor system performance and resource health.

---

## Information Summary

| Component | Description |
|---|---|
| AWS Service | Amazon CloudWatch |
| Dashboard | Central monitoring interface |
| Metrics | Performance data collected from AWS resources |
| Alarms | Notifications triggered when configured thresholds are exceeded |
