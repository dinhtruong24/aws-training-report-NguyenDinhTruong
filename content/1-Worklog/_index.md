Throughout the internship program "The First Cloud AI Journey", I have successfully completed the 12-week AWS learning path (from May 4, 2026, to July 25, 2026). This learning roadmap systematically combines theoretical knowledge, hands-on labs, and real-world project implementation with the detailed weekly log below:

**Week 1:** [Getting Started with AWS and Core AWS Services:](1.1-week1/) I focused on building a solid foundation by creating an AWS account, activating the Free Tier, and configuring AWS Budgets for cost management. I then practiced access control with AWS IAM, built an Amazon VPC, launched Amazon EC2 virtual servers, and used AWS Cloud9 and the AWS CLI to get familiar with the command-line interface.

**Week 2:** [Database and application deployment:](1.2-week2/) I worked with Amazon RDS (MySQL) and Amazon DynamoDB (NoSQL). Next, I deployed a web application on Amazon Lightsail, configured an Auto Scaling Group combined with an Elastic Load Balancer for automatic scaling and high availability, and learned how to package applications using Docker to run on EC2.

**Week 3:** [Security and Surveillance:](1.3-week3/) I focused on configuring security services, including IAM Identity Center (SSO), AWS Security Hub, AWS WAF, IAM Permission Boundaries, Amazon GuardDuty, and data encryption with AWS KMS. I also practiced real-time system monitoring by setting up metrics, alarms, and dashboards in Amazon CloudWatch.

**Week 4:** [Application Modernization & Serverless Computing:](1.4-week4/) I dove deep into hands-on practice with Serverless technologies like AWS Lambda and Amazon API Gateway to build basic microservices. I also learned the core principles of CI/CD in DevOps and utilized the AI assistant Amazon Q Developer to support code optimization and automate the software development lifecycle.

**Week 5:** [Container services:](1.5-week5/) I enhanced my Docker skills and practiced deploying applications with container clusters on Amazon ECS (running on both Fargate and EC2 launch types). I explored the core concepts of Amazon EKS, while setting up a CI/CD pipeline to automate packaging and rolling out containerized applications to a production-like environment.

**Week 6:** [Data and Analytics:](1.6-week6/) I practiced advanced configurations for Amazon RDS and Amazon DynamoDB to optimize query performance. Afterward, I researched and built practical data pipelines by exploring large-scale data processing services such as Amazon Kinesis and AWS Data Pipeline.

**Week 7:** [Artificial Intelligence/Machine Learning on AWS:](1.7-week7/) I explored the machine learning platform Amazon SageMaker and pre-trained AI services including Amazon Rekognition, Polly, and Textract. Finally, I practiced integrating Generative AI into applications by leveraging Amazon Bedrock and Amazon Q to solve simple real-world use cases.

**Week 8:** [FCJ Track & AWS Organization:](1.8-week8/) I focused on researching enterprise-scale multi-account management strategies and security governance using AWS Organizations combined with advanced IAM Identity Center configurations. Meanwhile, I completed other advanced specialized topics within the core FCJ training curriculum.

**Week 9:** [Review and consolidate knowledge:](1.9-week9/) I dedicated time to reviewing challenging technical topics and completing unfinished practical labs. Additionally, I began aligning and consolidating my system architecture against the 6 core pillars of the AWS Well-Architected Framework.

**Week 10:** [Review and consolidate knowledge:](1.10-week10/) I continued to tackle complex technical problems and finalized all remaining practical labs. I further optimized the deployed infrastructure in accordance with the operational excellence, security, and cost optimization standards of the AWS Well-Architected Framework.

**Week 11:** [Practical project:](1.11-week11/) I built a comprehensive AWS application integrating serverless technology, containers, databases, monitoring, and security. This project allowed me to apply all the knowledge gained during my internship to an end-to-end solution.

**Week 12:** [Final review and assessment:](1.12-week12/) I focused on finalizing my internship report to summarize my learning journey and technical achievements. I conducted a personal performance self-assessment, designed the presentation slides, and thoroughly prepared the live demo script to get ready for the final project defense before the evaluation committee.
